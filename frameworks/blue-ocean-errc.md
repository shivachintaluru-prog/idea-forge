## When to use
The market or solution space has a well-established set of dimensions everyone competes on. Use the ERRC grid (Eliminate, Reduce, Raise, Create) to systematically attack the conventional dimension set and find positioning territory competitors aren't fighting in.

## The technique
1. List 6-10 dimensions that current solutions in the space ALL compete on (e.g., for ride-sharing: speed, price, driver-quality, car-quality, app-polish, tipping options, coverage-area, surge-pricing-transparency, in-app entertainment, loyalty rewards).
2. For each dimension, force a placement in one of four cells:
   - **Eliminate**: get rid of this dimension entirely. What if we just don't compete on it?
   - **Reduce**: take this dimension well below industry standard.
   - **Raise**: take this dimension well above industry standard.
   - **Create**: introduce a NEW dimension nobody is competing on.
3. The ERRC grid forces a configuration that no current competitor is. Generate 3-5 ideas that operationalize the configuration.

## Anti-mode-collapse prompts
- Most dimensions get "raise" by default — that's mode collapse. For each dimension, explicitly consider if it should be ELIMINATED. (You can't compete on price if price doesn't exist as a dimension.)
- The Create cell is the highest-value but most-resisted cell. Generate at least 2 Create ideas — entirely new dimensions of competition.
- Reject configurations that are "moderate" everywhere. The point is to make sharp choices that produce a recognizably different shape.

## Example application (synthetic)
Problem: "Re-enter the meal-kit market."

Dimensions current competitors compete on: variety, ingredient-quality, packaging, recipe-difficulty, delivery-frequency, price, customization, dietary-options, brand-aesthetic, convenience.

ERRC:
- **Eliminate**: customization (every box is identical for everyone in the country that week — solidarity meal-kit)
- **Reduce**: variety (5 dishes per quarter, deep-rotating)
- **Raise**: ingredient-quality (sourced from specific named farms, traceability)
- **Create**: a new dimension — "shared mealtime" — every subscriber cooks the same dish on the same evening; livestreamed cooking guidance; the meal-kit is also a synchronous social event.

The configuration "no customization, low variety, high quality, synchronous social experience" is a different category from current meal-kits — Blue Ocean territory.

## Output format
- Dimension list
- ERRC grid (4 cells, dimensions placed)
- 3-5 ideas operationalizing the configuration
