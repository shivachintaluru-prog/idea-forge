## When to use
A problem statement carries hidden assumptions that constrain the solution space. Use False Faces to expose and reverse those assumptions, opening territory the original framing forbade.

## The technique
1. List 3-5 IMPLICIT assumptions buried in the problem statement. Look especially for the ones you'd never think to question — they sound like obvious truths.
2. For each assumption, write its literal REVERSE.
3. Pick the most provocative reversal — the one that makes you most uncomfortable.
4. Rewrite the problem statement assuming that reversal is true.
5. Repeat with the next-most-provocative reversal.

## Anti-mode-collapse prompts
- "List the assumptions you'd CHARITABLY grant — the ones you'd never argue with. Those are the ones to flip."
- Before reversing, ask: "If I bet $100 this assumption is true, what's the bet really riding on?"
- Reject reversals that produce trivial or absurd statements with no alternative meaning. Each reversal should suggest a different *direction*, not a contradiction.

## Example application (synthetic)
Problem: "Make a faster onboarding flow for first-time users."

Assumptions: (a) onboarding should be fast; (b) speed correlates with completion; (c) the goal is to get users to the app's main screen.

Reversals: (a) onboarding should be slow; (b) speed has no relation to completion; (c) the goal is to get users to leave reassured, not "in."

Reframe from (a): "Make an onboarding flow whose deliberate pacing teaches the value before exposing the interface."

## Output format
- 3-5 assumption reversals
- 1-3 rewritten problem statements (one per accepted reversal)
- Forward to Phase 3 as candidate reframes
