## When to use
Time-boxed forced-quantity sketching: produce 8 ideas in ~8 units of compressed effort. The forced quantity defeats the perfectionist filter; the time box defeats the "let me think about which is best" instinct. Best for problems with a tangible artifact (UI, layout, narrative arc) where rapid sketching can capture variation.

## The technique
1. Set a tight upper bound on idea generation — 8 ideas in 8 short bursts. Each burst is ~1 minute (or 1 paragraph if textual).
2. The first idea is allowed to be obvious. Ideas 2-8 must explicitly NOT repeat any structural element of the prior ideas.
3. After each idea, write its ONE-LINE LABEL and move on. Do NOT polish, expand, or evaluate.
4. After all 8 are produced, scan for the 1-2 that most surprised you. Those are the harvest.

The tight loop forces breadth; the no-repeat rule forces structural variety; the no-polish rule defeats the convergence-toward-quality instinct.

## Anti-mode-collapse prompts
- The first idea is permitted to be obvious — it discharges the convergent default. Write it down and move on.
- After idea 4, the temptation to "develop" rather than continue is strong. Resist; finish all 8 before any development.
- The no-repeat rule is structural, not topical. "Two-column layout with a sidebar" and "two-column layout with a top-bar" are repeats — same structure. Break the structure each time.
- "Crazy" in the name is the operative word: ideas 6-8 should genuinely feel uncomfortable, not just slightly different.

## Example application (synthetic)
Problem: "Onboarding for a new AI tool."

8 ideas (1-line labels):
1. Standard 3-step wizard.
2. Single-screen with all fields and a "magic populate" button that fills sensible defaults.
3. No onboarding — drop user into the main UI with annotated overlays they can dismiss one-by-one.
4. Onboarding as a 60-second video the user must speak through ("describe what you want to do" — the AI listens).
5. Reverse onboarding: user uploads a sample of their own work, the tool ONBOARDS ITSELF to the user.
6. Onboarding as a multiplayer experience — connect with a stranger also onboarding, complete a shared task together.
7. Onboarding rejected entirely — the only way in is via a referrer who has already onboarded; user inherits a context.
8. Onboarding as a fictional dialogue — user converses with an in-product character who claims to be from the future and explains how they'll use the tool.

Ideas 5 and 7 are the harvest — both genuinely uncomfortable, both produce different downstream products entirely.

## Output format
- 8 one-line ideas, in order of generation
- 1-2 highlighted as the surprising harvest
